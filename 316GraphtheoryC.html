Overview

This applet focuses on route optimisation methods in the AHL 3.16B syllabus. It is built around weighted, undirected graphs and helps students see how practical road networks can become route problems that can be solved step by step.

Chinese postman

Find the shortest closed route that uses every edge at least once. The key idea is to look at the odd vertices and work out the smallest extra distance needed to make all degrees even.

Least-distance completion

Turn a practical road network into a complete weighted graph by replacing missing direct links with shortest-path distances. This creates the table needed for classical TSP methods.

Travelling salesman problem

On a complete weighted graph, compare a nearest neighbour upper bound with a deleted vertex lower bound. For small graphs, you can also check the exact best Hamiltonian cycle.

Suggested classroom flow: load a sample graph, discuss what the graph represents, predict what should happen, then run the algorithm and compare the output with student reasoning.

Included

Chinese postman for up to 4 odd vertices, least-distance tables, nearest neighbour, deleted vertex lower bound, and an optional exact TSP check for small graphs.

Not the main focus here

General walk/trail/path/cycle classification, Euler/Hamiltonian existence as a main topic, and MST as a main topic.

Definitions and Method

Odd vertex

A vertex with an odd number of incident edges. Chinese postman depends on these because an Eulerian circuit needs every degree to be even.

Least-distance table

A table of the shortest distances between every pair of vertices. This lets us treat a road network as a complete weighted graph.

Upper bound

A route length we know can be achieved. Nearest neighbour gives an upper bound because it constructs an actual Hamiltonian cycle.

Lower bound

A value the true optimal route cannot go below. Deleted vertex gives a lower bound, but not usually the exact answer.

Nearest neighbour

Start at one vertex, go to the nearest unvisited vertex, repeat, then return to the start. It is quick but not always optimal.

Deleted vertex method

Delete one vertex, find an MST on the remaining vertices, and add the two smallest edges from the deleted vertex. Repeat and take the largest result.

Method summary: Chinese postman is about repeating as little distance as possible. TSP is about visiting each vertex exactly once in a Hamiltonian cycle with minimum total weight.

Student Challenges

Challenge 1

Load a graph with 2 odd vertices. Predict which extra path will be repeated before you run the Chinese postman solver.

Challenge 2

Load a graph with 4 odd vertices. Write down all three pairings and decide which one should give the smallest extra cost.

Challenge 3

Load the incomplete road network. Use the least-distance matrix to explain how a non-complete graph becomes a complete TSP table.

Challenge 4

On the complete weighted graph, run nearest neighbour and deleted vertex. Compare the upper and lower bounds and explain what the gap means.

Challenge 5

Change one edge weight and see how the postman length, upper bound, and lower bound respond. Which method is most sensitive to the change?

Discussion prompt

Why is a Chinese postman route allowed to repeat edges, but a travelling salesman route must instead visit each vertex exactly once?
