AHL 3.15 syllabus focus

Adjacency matrices

Turn a graph into a matrix and read connection information from rows and columns.

Walks and Aᵏ

Use powers of the adjacency matrix to count the number of walks of length k between two vertices.

Weighted adjacency tables

Use weights to represent things such as time, cost, or distance.

Transition matrices

Construct a probability matrix from a graph and connect this to random movement and PageRank ideas.

Core result: If A is an adjacency matrix, then the entry in row i, column j of Aᵏ gives the number of walks of length k from vertex i to vertex j.

What students should be able to do

Construct an adjacency matrix from a graph.

Interpret entries in A, A², A³, and higher powers.

Count walks of exactly length k and at most length k.

Construct a weighted adjacency table.

Construct a transition matrix for a connected or strongly connected graph.

Important links

Weights may represent distance, cost, or time.

Transition matrices link naturally to Markov chains.

Directed graphs provide a simple model for Google PageRank.

Transition Lab

Transition matrix P

Each row gives the probabilities of moving to the next vertex in one step.

Random walk / simple PageRank idea

What to notice

For a directed graph, divide each row of the adjacency matrix by the row sum to make a transition matrix.

For an undirected graph, the same idea works using degree.

To use this comfortably, the graph should be connected or strongly connected, so movement does not get stuck in isolated parts.

PageRank connection

Think of a web user following links from page to page. Repeated multiplication by the transition matrix shows how attention can settle onto certain pages.

This applet shows a simple random-walk version to build intuition.

Student Activities

Activity 1: Build a graph with 4 vertices and write down its adjacency matrix before checking the applet.
Activity 2: Use the sample undirected graph. Predict the number of 2-step walks from A to C, then check A².
Activity 3: Compare A² and A³. Which one counts longer walks, and how can you tell?
Activity 4: Use walks of length at most 3. Find two vertices with more than one possible route between them.
Activity 5: Turn on weights and build a road network. Explain what each weight represents and read the weighted adjacency table.
Activity 6: Use a directed graph and construct its transition matrix. Explain why each row adds to 1.
Activity 7: Run the simple PageRank tool on a directed graph. Which vertex ends up with the highest long-run share?

Reflection prompts

Why does A² count walks of length 2 rather than just showing direct edges?

Why are walks of length at most k found by adding several matrix powers together?

What does a large entry in a transition or rank vector mean in context?
